OMI

  • general
    • OMI extras compared to intouch:
      • screen profiles; determines where, meaning on which display screen
      • layouts; define how you want to organize content
      • archestra applications; are prebuilt, special-purpose collections of one or more controls
    • an OMI app is called an ViewApp
    • provides multi-monitor setups
    • is provided as a part of Application Server and is not exclusive with intouch
    • is launched through application manager
  • features
    • built-in navigation; that enables users to browser for and display graphics in a running ViewApp. Based on the plant model of the galaxy, but can be customized
    • integrated security; that is integrated with the galaxy
    • single sign-on service; also integrated with the galaxy
    • native support for archestra symbols; that is, you can use archestra symbols in your ViewApp
    • faceplates; symbols enabled for faceplate compatibilty mode allow users to manage similar symbols using a single symbol
    • touch and multitouch gestures; pan, zoom, swipe, close and so on.
    • resolution independence; eliminates the need to configure the display resolution
    • software developer kit; lets you build custom applications. Uses dotnet
  • screen profiles, layouts and panes
    • screen profiles:
      • defines the physical characteristics of one+ client workspace monitor, including screen arrangement
    • layout:
      • one+ rectangular area of content for a screen. shows runtime information. every rectangular area is called a pane.
    • pane:
      • a sub-area of a layout that serves as a view-container. can include items like navigation trees, other layouts and so on.
  • archestra apps
    • an archestra app in a collection of one+ controls and is primarily developed with WPF (winform/html5 is also available)
    • AA apps can be used as content in a pane
    • included apps:
      • AlarmApp shows current and historical alarms
      • ContentPresenterApp shows graphic items from one or more navigation items
      • HamburgerApp controls the movement of a selected slide-in layout pane. three horizontal panes
      • HistoricalTrendApp plots historian data to a trend chart.
      • InSightApp shows historical operational data from historian
      • MapApp shows geo-data on a map
      • NavigationApp sets the current navigation item. it includes:
        • NavBreadcrumbControl shows the user-selected sequential path
        • NavigationTree shows the hierarchical organization of items
      • TitleBarApp provides functional components to the title bar (home, login, language, keyboard)
      • WWWebAppControls:
        • SharePoint integrates sharepoint and is primarily used as document management
        • WebBrowser is a fully featured web browser.
